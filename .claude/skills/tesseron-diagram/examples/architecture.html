Tesseron Architecture

Three processes, two protocols - JSON-RPC/WS between app and gateway, MCP stdio between gateway and agent.

Your web app

  • Hosts the action handlers and the real state they mutate.
  • Uses @tesseron/web, /server, or /react.
  • Connects once, declares once, runs on user's machine.

Tesseron gateway

  • Bundled into the Claude Code plugin; auto-spawned.
  • WebSocket server on 127.0.0.1:7475 + MCP stdio bridge.
  • Routes tools/call to actions/invoke and back.

AI agent

  • Any MCP-capable client - Claude Code, Cursor, Desktop.
  • Sees the app's actions as native MCP tools.
  • Never knows WebSocket exists.
