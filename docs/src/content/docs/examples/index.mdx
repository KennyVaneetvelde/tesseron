## Recommended reading order

1. **[vanilla-todo](/examples/vanilla-todo/)** - plain DOM, no framework. The SDK's builder API with nothing in the way.
2. **[node-todo](/examples/node-todo/)** - the same action declarations on Node. Proves nothing's tied to the browser.
3. **[express-todo](/examples/express-todo/)** - adds a REST API next to Tesseron. Shows the "same state, two channels" pattern.
4. **[react-todo](/examples/react-todo/)** - hooks-based integration.
5. **[svelte-todo](/examples/svelte-todo/)** - mutation via `$state` runes.
6. **[vue-todo](/examples/vue-todo/)** - mutation via `ref().value` / `computed()`.

## Running any of them

```bash
git clone https://github.com/KennyVaneetvelde/tesseron
cd tesseron
pnpm install
pnpm --filter <example-name> dev
```

Then claim the session from your agent - see the [quickstart](/overview/quickstart/) if you haven't already.
